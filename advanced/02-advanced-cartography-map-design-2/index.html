Map Themes

A map theme is a snapshot of the current map legend that records:

  • the layers set as visible in the Layers panel and for each visible layer:
  • the reference to the style applied to the layer
  • the visible classes of the style, (i.e. the layer checked node items in the Layers panel. This applies to symbologies other than the single symbol rendering)
  • the collapsed/expanded state of the layer node(s) and the group(s) it’s placed inside

Map theme options can be found in the Layer panel via the  button.

To create a map theme:

  1. Check a layer/s you want to show.
  2. Configure the layer properties (symbology, diagram, labels, etc).
  1. Expand the Style ‣ menu at the bottom of the Styling Panel.
  2. Click on Add… to store the settings as a new style embedded in the project.
  1. Repeat the previous steps as necessary for the other layers.
  2. If applicable, expand or collapse groups or visible layer nodes in the Layers panel.
  3. Click on the  Manage Map Themes button on top of the panel and click Add Theme…
  4. Enter the map theme’s name and click OK.
  5. Your themes are listed under the  button.

NOTES:

A map theme does not remember the current details of the properties: only a reference to the style name is saved, so whenever you apply modifications to the layer while this style is enabled (eg change the symbology rendering), the map theme is updated with new information.

You can create as many map themes as needed.

Map themes allow you to quickly switch between different configurations of the layers in your map. Also, since map themes are available in the Print Layout, they allow you to create map items based on the themes and independent of the current rendering or appearance of the main canvas. This is extremely useful when adding multiple maps in a layout (e.g. insets, etc.).

Layout Basics

The Print Layout window can be accessed via Project -> New Print Layout in the Menu bar or CTRL + P. QGIS also has a layout manager (Project -> Layout Manager) that allows the user to manage (add, delete, duplicate) print layouts related to a project.

NOTE:

A Print Layout is saved together with the QGIS project and is not available outside of the QGIS project where it is used. However, QGIS has templates which can be used for different projects. A plugin, Layout Loader, can also be used for applying layout templates to your maps.

Page Setup

Upon opening the Print Layout window and before adding elements and components to the map, it is essential to set up the layout of your map first. This includes deciding the size, orientation, and background color (if any) of your layout or printing material. This option can be accessed by Right clicking on the layout canvas -> Page Properties.

To view the finished map, open the Layout Basics.qgz project inside the 03 Advanced Cartography & Map Design II geopackage.

The Print Layout provides a Layout/Print canvas where we can add different components such as maps, legends, scales, etc.

The buttons/commands for adding these components can be found on the Toolbar on the left of the print canvas. Adding a component to the print canvas usually involves clicking one of the buttons and dragging on the print canvas where you want the said component to be placed.

The options to modify the properties of these components (e.g. size, color, font, etc.) can be found on the Panel to the right of the print canvas.

Adding a continuous/gradient legend

Exercise 5. Adding a continuous/gradient legend

Currently, there is no direct way to add a continuous legend (similar to those used for rasters) in the QGIS Print Layout. There is, however, a simple workaround to doing this.

In this exercise, we’ll add a continuous legend in the print layout by using a rectangle map element and some gradient styling.

  1. Load the SRTM_Albay raster. Rename it as Elevation (m) in the Layer Panel.
  2. Apply a Singleband Pseudocolor symbology to the raster. Select any color ramp but try to use an Equal Interval Mode.

To view the finished map, open the Working with Legends II.qgz project inside the 03 Advanced Cartography & Map Design II geopackage.

  1. Open a Print Layout. Add the map, legend, and some other components.

  1. On the Item Properties of the legend, edit the following:
  1. Symbol
  1. Symbol height: 0.5 mm
  2. Uncheck Draw stroke for raster symbol
  1. Spacing
  1. Legend Items
  1. Space between symbols: 8mm
  1. You may change the values above to your liking.

  1. Add a rectangle to your layout using the  button. Make sure that the rectangle covers the height of your legend (i.e. from top to bottom). You can use guides for this.

  1. Apply a gradient style to the rectangle by editing its style to a gradient fill. Style -> Configure Symbol -> Gradient Fill.
  2. Edit the Gradient fill to correspond to the color ramp you used.
  1. If you have 5 categories, you should have 5 stops. Each stop/category should have the same color as the one in the symbology.

  1. Move the rectangle so it covers the original legend.

Generating Maps with Atlas

The QGIS Atlas

QGIS Atlas allows you to automatically create a series of maps using a single layout. A properly configured QGIS Atlas will save you countless hours.

Exercise 7. Generating provincial COVID-19 maps per region using QGIS Atlas

In this exercise, we’ll generate a map for each region in the Philippines showing the number of active COVID-19 cases for each province/district in the region.

  1. Load the stompcovidph_provinces vector layer and style it. For this exercise, we will use a rule-based symbology to style our layer. The top rule (Active Cases), filters our layer so that only the geometries within the atlas geometry (in this case the regions) will be shown. The refined rules under the top rule is a graduated symbology similar to those done in the previous exercises.

To view the finished map, open the QGIS Atlas.qgz project inside the 03 Advanced Cartography & Map Design II geopackage.

  1. It’s okay if the provinces layer isn’t shown on the map canvas upon applying this symbology. That is to be expected since we have yet to define our Atlas and the symbology depends on @atlas_geometry.

  1. Let’s configure our atlas. Open a new Print Layout and add a map.
  2.  Click  on the top toolbar to open the Atlas Settings.

  1. Configure your Atlas with the following:
  1. Configuration
  1. Coverage layer: stompcovidph_regions
  2. Check Hidden coverage layer
  3. Page name: region

  1. Modify the Item Properties of the Map Item and check Controlled by Atlas.

  1. Click  on the top toolbar to preview your atlas. The Atlas Toolbar shown below allows you to see the different maps in your atlas. Try to click the arrow buttons and see what happens to the map.

  1. Next, we can add a data-defined title to our maps that changes values depending on the region attribute of the current atlas coverage.

  1. We can also add a legend for our Atlas maps.

  1. You can add other map items/elements and configure them as you like. Once you’re happy with your Atlas, you can Export your atlas as images by clicking . Depending on how you configured your atlas, this may result in a single image or multiple images (1 per map).

  1. Note that you can add other maps and styles in the map canvas and they will also reflect in the Atlas. For example, if you duplicate the active cases layer and style it as a dull version of the original symbology, it can help add context to the Atlas maps.

Using Reports

QGIS Reports is similar to the Atlas capability but with more functionalities. With Reports, you aren’t constrained to just creating maps. You can add sections of texts, images, etc. Reports can even have other reports nested inside them. You can think of Reports as Atlas on steroids.

Since using Reports can get overly complicated sometimes, in the next exercise, we’ll show a common use-case for reports: most common use case: a report consisting of pages that, if you were to tape them together, would comprise a single large map. This next exercise is based on Section 10. Creating Automated Reports of QGIS Map Design, second edition.

QGIS Reports

Exercise 8: Creating a Report
  1. Open the QGIS Reports project from the 03 Advanced Cartography & Map Design II geopackage.
  2. Create a new report via Project -> New Report. This will open the Report Layout which is similar to the Print Layout.

To view the finished map, open the QGIS Reports.qgz project inside the 03 Advanced Cartography & Map Design II geopackage.

  1. Check Include report header and click Edit. A canvas will appear where you can edit said header. Add a title and a legend.

  1. Now let’s add the maps for our report. Click the plus sign and select Field Group Section. This will add a new section in our report. This section will contain our maps.

 

  1. Edit the Field Group Section.
  1. Set Layer to Grid and Field to id. The grid will act similar to the coverage layer in the Atlas.
  2. Check Include body and click Edit.
  3. Add a map and in its Item Properties, check Controlled by Report (similar to Controlled by Atlas)

  1. Add a Static Layout Section at the end of the Report that shows the map in its entirety.

  1. You can then print or save the report as an image or PDF.
