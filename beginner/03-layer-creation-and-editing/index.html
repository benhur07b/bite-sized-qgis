Introduction

This workbook is designed to teach beginners how to work and edit layers in QGIS, particularly vector layers. It also includes several activities on digitizing and georeferencing.

By the end of the workbook, you should know: when to use temporary and virtual layers; how to digitize vector data; how to georeference raster data; how to load OSM data in QGIS; and how to map tracts of land from titles and technical descriptions.

What you should already know

Since this is a beginner workbook, no previous knowledge of GIS or QGIS is required.

Creating new geospatial data

Geospatial data is everywhere and a lot of it is already in digital form ready to be loaded and used in a QGIS. However, not all geospatial data have been digitized. Prior to computers being ubiquitous parts of human life, hard-copy paper maps were the primary form of storing and showing geospatial data. Fortunately, it is possible to liberate the geospatial data from these paper maps through the process called digiting.

In the next few exercises, we will digitize some parcels from a non-georeferenced scanned map by doing the following steps:

  1. Creating a vector layer to store the data to be digitized.
  2. Opening the QGIS GDAL Georeferencer.
  3. Loading the non-georeferenced source map in the Georeferencer.
  4. Georeferencing the source map.
  5. Digitizing the parcels.
  6. Saving / Exporting the data.

Before diving straight into georeferencing and digitizing, we’ll have a brief digression into the concepts of layers in memory and virtual layers.

Layers in memory

Sometimes, we don’t need or want to physically save the layers we use in our analysis in our computers, especially if they are just temporary.

QGIS understands this and allows users to create temporary / scratch layers. These layers can be used similar to regular vector layers but do not need to be permanently saved into a file in our storage drives because QGIS stores them in memory or RAM. The downside of this is that QGIS “forgets” these “layers in memory” when it is closed. To remedy this, we can use the Memory Layer Saver plugin to allow our temporary layers to be persistent. When saving a QGIS project that uses temporary layers, it is good practice to run/activate the Memory Layer Saver plugin first.

Most outputs of processing algorithms are stored as temporary layers by default.

Temporary layers are shown in the Layer Panel with this icon .

Virtual layers

A virtual layer is a special type of vector layer that is created “on the fly” as a result of a query or data from another layer.

For example, we can create a virtual layer that creates a buffer around another feature layer. Whenever a new feature is added to the source feature layer, the virtual layer will be updated accordingly.

Currently, virtual layers don’t seem to work with source layers in memory.

Exercise 1: Working with virtual layers
  1. Open QGIS.
  2. Load a satellite image or satellite basemap via XYZ tiles or QuickMapServices plugin (e.g. Google Satellite).
  3. Create a new geopackage layer with  in the Data Source Manager toolbar. Name it Trees, add a radius (integer) and species (text) fields, and use a projected CRS (e.g. EPSG:3857)

When you are prompted that the file already exists, select Add New Layer.

  1. Toggle editing on Trees by right-clicking on it -> Toggle editing or clicking  in the Digitizing Toolbar.
  2. Select an area on the satellite image that has trees.
  3. With Editing toggled on, enable adding points to the Trees layer by clicking   (CTRL + .).
  4. Digitize individual trees by clicking on their location and adding the attribute information. Do this for 4 trees. For this exercise, can just guess the information.

  1. After adding the points, don’t forget to save your edits by clicking  in the Digitizing toolbar before toggling the Editing off.

  1. Add a virtual layer by clicking  in the Data Source Manager toolbar.
  2. Set Layer name to Tree Canopy. Import Trees. Add the following query:

select fid, buffer(geometry, radius), species from Trees

  1. Add a new point then refresh the map display (e.g. zoom, pan) to see the update in the virtual layer. What did you notice?

Virtual layers are dynamic. This means that when the base/source layer is updated, the virtual layer is updated as well. This can save space and reduce data duplication since the virtual layer data will simply get data from the base layer without needing to save anything to your disk although there can be some performance overhead if complex queries or operations are used to define the virtual layer.

SQL-like constructs are used to define the virtual layer.

Virtual layers are dependent on other layers so it is important that the base layers are not moved or renamed.

When the base layer is updated, you need to refresh the map display by panning or zooming the map to show the virtual layer update.

Exporting layers

Exporting layers (or saving layers to files) is straightforward in QGIS. Simply right-click on the layer -> Export  to view the export options for the layer.

You can perform coordinate conversion while exporting the layer by choosing a different CRS from the layer’s CRS.

Save Selected Features as is a simple way to create a subset of a vector file based on a selection.

For Filtered layers, only the filtered features (those appearing on the map canvas) are exported.

Exercise 2: Exporting a Layer
  1. Export your Tree Canopy to the GeoPackage you created earlier by right-clicking in Tree Canopy in the Layer Panel -> Export-> Save Features As…
  2. Name the Layer as Tree Canopy.

Georeferencing rasters in QGIS

GDAL Georeferencer

One way to georeference data in QGIS is via the GDAL Georeferencer which is a Core Plugin. Go to the Manage and Install Plugins dialog and make sure that the GDAL Goereferencer is activated (i.e. there is a check mark next to the name of the plugin).

Exercise 3. Georeferencing with the GDAL Georeferencer
  1. Check the CRS of your QGIS project by looking at the lower right-hand corner of the QGIS UI along the Status Bar. Ensure that it is EPSG:4326. If not, set the CRS to EPSG:4326.
  2. Just to be safe, save your project and name it 03 Layer Creation & Editing. (HINT: You can also save your project directly into the GeoPackage you created earlier by going to: Project ->  Save to -> GeoPackage.)
  3. Open the Georeferencer via Raster -> Georeferencer. A new Georeferencer window should appear.

  1. Open the Georeferencer via Raster -> Georeferencer. A new Georeferencer window should appear.
  2. Click on  on the Georeferencer window to open the raster that we will georeference (i.e. Zoning Map.png)

  1. What is the CRS of the map? Take note of this as this is important.
  2. Click  to set the Transformation Parameters for georeferencing.
  3. You can select from several Transformation types and Resampling methods varying from the simple to complex. In our case, a simple Linear transform with Nearest neighbor resampling should suffice. Set the Target CRS to the CRS of the map.

  1. Click OK.

When selecting the transformation parameters, the things to consider are:

  • the map’s complexity and distortion (regular-shaped maps normally only need to use simple transformation parameters)
  • the number of GCPs that you can get on the map -- the more complex the transformation type, the more GCPs are needed to get good results.
  • the distribution of GCPs on the map -- poor GCP distribution results in more distortion especially in higher order transformation equations.
  • more (or more complex) is not always better.

  1. The next step is to add GCPs. GCPs or Ground Control Points refer to points on the map whose coordinates are known. In this case, we are lucky that the map has coordinates found on its border.
  2. Click on the Add point button . If you make a mistake or want to redo a point, you can click the Delete point button  and click on the GCP to delete or you can use the Move point button  to move the GCP.
  3. Select a point on the map with known coordinates like the grid intersections.

  1. In the Enter Map Coordinates dialog, type the coordinates as shown on the map.

  1. Repeat 3 more times until you have 4 GCPs. The GCP table will show the computed residuals for the transformation. It is always best to keep these residuals to a minimum.

  1. You can save the GCPs into a file by clicking . This will allow you to load the GCPs in the future.
  2. Click  to Run the georeferencing.
  3. QGIS should prompt you when the georeferencing is complete and load the layer in QGIS (if you checked the option in the Transformation Settings).
  4. You can load a basemap layer in QGIS and update the transparency of the georeferenced raster to see if it is in the correct place.

There are two ways to add GCPs when georeferencing in QGIS. The method shown above is useful when you have easily identifiable points or locations in the map with known coordinates that you can type. In cases where this is not the case, the method to use is:

  1. Load an georeferenced map (target) in QGIS.
  2. When adding GCPs for the map to be georeferenced (source), click  on the Enter Map Coordinates dialog and select or match the points or features on the source map to the target map.

Freehand Raster Georeferencer

Another option you can try for georeferencing is the Freehand Raster Georeferencer plugin. It’s useful for quick georeferencing and works best with maps that only need some translation, rotation, and scale to georeference.

The plugin works by loading the map to georeference in the QGIS canvas and allowing the user to freely move, rotate, and scale the map until it is georeferenced or the features between the map and a basemap source match. As such, the plugin has commands for adding, moving, rotating, and scaling the raster to be georeferenced.

Digitizing new data

Digitizing Settings

Digitizing settings can be found under Settings -> Options -> Digitizing tab. Here the user can set options for feature creation, snapping, and other useful parameters when digitizing data.

Digitizing Toolbar

The Digitizing toolbar contains the commands that will allow you to digitize data. It can be activated under View -> Toolbars -> Digitizing Toolbar.

In order to digitize data, make sure that Editing is enabled on the layer. This is done by clicking the Toggle editing button  while the layer is selected.

Advanced Digitizing Toolbar

The Advanced Digitizing Toolbar can be activated via Views -> Toolbar -> Advanced Digitizing Toolbar. This toolbar has functions for adding rings, adding parts, moving and rotating polygons,, and other advanced digitizing options.

Digitizing Tools Plugin

The Digitizing Tools plugin adds more functions to digitizing in QGIS and subsumes some of the features of the Basic and Advanced Digitizing toolbars.

Raster Tracer Plugin

The Raster Tracer plugin allows the user to automatically trace line features in a raster map which is useful for tracing things like contours. It works by setting a color that the plugin will trace. The user clicks on a start and end point. The plugin then tries to trace a line along pixels between the start and end point with the same color as the one set by the user.

Getting OSM data into QGIS

OpenStreetMap (OSM) is the free editable map of the world. Data from OSM can directly be loaded in QGIS. This is useful since OSM data has an enormous amount of data ranging from POIs, roads, building footprints, etc.

The QuickOSM Plugin

Loading OSM data in QGIS is fairly straightforward using the QuickOSM plugin although some familiarity with OSM tagging conventions is needed. For OSM power users, QuickOSM also allows for custom Overpass queries to be used.

Mapping tracts of land from technical descriptions and land titles

In mapping tracts of land from technical descriptions and land titles, the most difficult part is getting the coordinates of the tie-point.

Azimuth and Distance Plugin

The Azimuth and Distance Plugin is one of the simplest means to map survey returns (e.g. bearings and distances).

It can be installed from the Manage and Install Plugins dialog and run via Plugins -> Topography -> Azimuth and distance.

Inputs to the plugin can be done manually in QGIS or through an input file (which is basically a text file that tells the plugin about what data to plot).

Exercise 7: How mapping land parcels and traverses with the Azimuth and Distance plugin works
  1. Set the options in the Options tab.
  1. Survey type: Boundary
  2. Angle type: Bearing
  3. Distance unit: Default
  4. Angle unit: Degree

  1. Set the CRS of the project to the correct CRS. Projected CRS (e.g. PRS92 Projected 51N, UTM Zone 51N, etc.) are preferred.
  2. Create or select a layer to hold the data to be mapped (optional). If a layer is not created, the data will be stored as a layer in memory.
  3. Go to the Drawing tab.

  1. Add the coordinates of the Starting Point. You can also select this from the map if needed.
  2. Add the bearing and distance parameters.
  1. Azimuth: Bearing in N/S DD MM SS.SS E/W
  2. Distance: Distance
  1. Click Add to Bottom or Insert Above depending on where the observation should be.
  2. Repeat until all observations or bearings and distances have been added.
  3. When adding a new observation, the line is automatically shown as a preview (purple dashed line).
  4. To draw the line / or create a vector layer in QGIS, click
  5. You can also import and export parameters via the Import and Export buttons.

A somewhat faster way to do this is by preparing the input file beforehand and importing it later. The import file of the plugin is simply a text file which can be opened with any text editor. The content are shown below:

angle=Bearing

heading=Coordinate_System

dist_units=meters

angle_unit=degree

startAt=289724.009576487;1620747.9768304361;90

survey=Polygonal

[data]

N 90 0 0 E;1000.0;90;None;None

S 45 12 13 W;300.0;90;None;None

The first few lines of the file correspond to the values provided in the Options tab. It also has information on the Starting Point (StartAt) of the survey.

The lines under [data] correspond to the observations or bearings and distances and follows the format: Azimuth, Distance, Zenith, Angle, Direction.

This file can then be edited to include all the observations in a survey and later imported into the plugin.

CoGo Plugin

The CoGo plugin is a more advanced albeit scalable method of mapping cadastres and tracts of land. It utilizes a Postgres + PostGIS database with QGIS to store location information of bearings and distances, boundary points, etc. It then uses this information to generate cadastral and parcellary maps.
