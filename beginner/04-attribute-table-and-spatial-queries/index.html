Introduction

This workbook is designed to introduce the attribute data and the different things that can be done with attributes: joins, selects, filters, etc. This workbook will also introduce spatial relationships and spatial queries or queries by location.

By the end of the workbook, you should know: how to work with attribute data; perform selects and filters using both attributes and location.

What you should already know

Since this is a beginner workbook, no previous knowledge of GIS or QGIS is required.

The Attribute table

If we go back to the discussion of the spatial data model, we know that it is composed of two components:

  • spatial features (geometry)
  • attributes

In the previous workbook, we focused more on creating and editing spatial features. Here, we’ll look at the other component of the spatial data model: attributes.

Exercise 1: Familiarizing yourself with the attribute table
  1.  Load the PHL_provinces vector layer inside the 04 The Attribute Table & Spatial Queries geopackage.
  2. Right-click on the layer -> Open Attribute Table or click F6 with the layer selected. What do you see? The attribute table shows the attributes for each feature in the layer. It also shows how many features are in the layer, how many are currently filtered, and how many are currently selected.

  1. You can think of the attribute table as similar to any table in a spreadsheet. You can select rows and cells, copy values, sort rows by column values, resize the columns, and even hide columns.
  2. In the attribute table, each row represents a feature in the vector layer and each column represents an attribute. Clicking on the headers (e.g. fid, province, etc.) sorts the rows based on the selected attribute. This is useful if you want to quickly find which features have the highest or lowest values (e.g. find the top and bottom 5 provinces in terms of population in 2015).

  1. Selecting a row in the attribute column also selects that feature in the map canvas.

  1. Right-clicking on the header row provides you options to manage and organize your columns which includes the option to hide and show columns which is useful for improving focus when working with the attribute table.
  2. Organize your columns and hide the fid, region and psgc_region fields.

  1. Other options that allow you to edit the content of the Attribute Table are found in the lower left hand corner of the table.

Attribute joins

Attribute joins refer to the act of joining together two tables based on a common column or attribute. This is very useful if you want to add attribute data to existing spatial data and one of the most important functions of GIS as it allows for non-spatial data to be made spatial.

For example, in the next exercise we will add COVID-19 case counts per province in our province vector file which would, in turn, allow us to map these case counts instead of just analyzing them numerically.

There are two ways to perform attribute joins in QGIS. The first is via the layer’s Properties -> Join and the second is via the Join attributes by field value processing algorithm. The first one can be considered a soft join since it temporarily adds fields from the join table to the target table which can be removed/unjoined any time. The second option can be considered a hard join since it creates a new table from the joined attribute tables.

Select and Filter by attributes

Difference of Select and Filter

Select and Filter in QGIS perform similarly. A select expression will work equally well as a filter expression. The main difference between the two is their interaction with a layer’s features.

A filter only shows the features that meet its criteria and temporarily disable or hide those that don’t. Because of this, running a processing algorithm on a filtered layer will only apply the algorithm on the filtered features (subset). This is also the case when exporting a filtered layer. Only the filtered layers are exported.

A select, meanwhile, does not modify the features shown by the layer but merely selects or highlights the features that meet the select criteria. By default, running algorithms on a layer with some selected features would still result in having the algorithm go over all of the layer’s features. However, QGIS provides the option Run on selected features only in most of its processing algorithms in case you only want to run your algorithm in the subset you selected. This is also true in exporting a layer with selected features. QGIS provides the option to Save Selected Features Only....

As with most parts of QGIS, the Select and Filter functionalities work with the QGIS Expression Engine which enables complex select and filter conditions to be made.

Field calculator

Sometimes, the attribute we want to observe is not explicitly part of the attribute table but could be calculated using the existing values. For these purposes, QGIS has a powerful Field Calculator that can be opened by clicking the Field Calculator button  in the Attributes Toolbar or in the attribute table itself.

Exercise 5: Computing recovery rate, case fatality rate, and cases per 100K
  1. Open the Field Calculator for the PHL_provinces layer.
  2. Create a recovery rate field
  1. Field type: Decimal number (real)
  2. Expression: 100*recoveries/confirmed

  1. Create a case fatality rate field
  1. Field type: Decimal number (real)
  2. Expression: 100*deaths/confirmed

  1. Create a per 100K field
  1. Field type: Decimal number (real)
  2. Expression: 100000*confirmed/pop2015

  1. Your attribute table should now have the fields you created.
  2. Don’t forget to save your edits to the attribute table.

Vector Statistics & Visualizations

You can check the summary statistics of attribute tables by clicking the button on the Attributes Toolbar or activating the Statistics Panel in Views -> Panel -> Statistics.

This shows some simple summary statistics for an attribute or field of a vector layer.

As with most native QGIS algorithms, you also have the option to compute the statistics for selected features only.

Group Stats plugin

The Group Stats plugin allows you to create compute pivot-table type statistics using the values in your attribute table. For example, we can compute for the sum, average, min, max per region.

Group Stats works by drag and dropping the fieldsto corresponding parts of the pivot table either as a row, column, or value.

Spatial relationships

Some spatial objects intersect, overlap, are contained by another spatial object, or totally disjoint from one another. In other words, spatial objects have a relationship with other spatial objects.

These spatial relationships or geometric predicates are based on The Dimensionally Extended nine-Intersection Model (DE-9IM) which is a topological model and standard used to describe the spatial relations of two regions or geometries.

The DE-9IM model describes a region as having an interior, boundary, and exterior. Geometric predicates are defined based on the interaction between these parts of the geometric object.

For example:

Two geometries are topologically EQUAL if their boundaries intersect and no part of the interior or boundary of one geometry intersects the exterior of the other.

They are DISJOINT if they have no point in common.

They TOUCH if they have at least one point in common, but their interiors do not intersect.

A CONTAINS B iff no points of B lie in the exterior of A, and at least one point of the interior of B lies in the interior of A.

They INTERSECT if they have at least one point in common.

Queries by location

Knowing about spatial predicates, we are able to create queries based on the relationship between different layers in QGIS.

Exercise 8. Select by location

One of the most common things we can do in a GIS that usually cannot be done in other information systems is doing analysis based on locations. For example, what if we want to select all persons at risk within a certain region?

  1. Load the Persons at Risk and PHL_provinces layers.
  2. We want to select all the persons at risk within Region V.
  3. Filter the PHL_provinces layer to include only the provinces of Region V (HINT: you can use region or psgc_region).
  4. Run Select by location (you can find this in the Location Bar).
  1. Select feature from: Persons At Risk
  2. Where the features: are within
  3. By comparing to the features from: PHL_provinces

You can then use this selection in other analysis, to count statistics, or for exporting as a new layer.

Exercise 9. Counting points in polygon

Another common thing to do when we have points, lines, and polygons is to count the number of objects within a certain region. In the next exercise, we will count the number of persons at risk within the different provinces.

  1. Load the Persons at Risk and PHL_provinces layers.
  2. To count the number of persons at risk for each province, run Count Points in Polygons (you can find this in the Locator Bar).
  1. Polygons: PHL_provinces
  2. Points: Persons At Risk
  1. A new Count layer should be added to the Layer Panel (if you did not save the output  as a file)

Exercise 10. Join by location

Joining by location means adding attributes to a vector layer based on its relationship with another vector layer. For this exercise, we want to tag the persons at risk with the recovery rate and case fatality rate information for their provinces.

What we’ll do is: For each point in the Persons At Risk layer, we will find the feature (i.e. province) in the PHL_provinces layer where this point is located within and join the recovery rate and case fatality attributes of the latter to the former.

  1. Run Join attributes by location.
  1. Base Layer: Person At Risk
  2. Join Layer: PHL_provinces
  3. Geometric predicate: within
  1. A new Joined layer layer should be added to the Layer Panel (if you did not save the output  as a file)

Exercise 11. Summarizing by area/location

Aside from selecting/filtering features by location and joining attributes by location, we can also summarize attributes by location. This is very useful when you want to summarize the attributes of a point or line vector per polygon. In the exercise below, we want to find out the mean, min, and max ages of the persons at risk for each province.

What we’ll do is: For each feature (i.e. province) in the PHL_provinces layer, we’ll get all the features (i.e. points) from the Person At Risk layer contained by the PHL_provinces  feature, summarize the age attribute of these Person At Risk features, and then add that as an attribute to the PHL_provinces feature.

  1. Run Join attributes by location (summary).
  1. Input layer: PHL_provinces
  2. Join layer: Persons At Risk
  3. Geometric predicate: contains
  1. Select the fields to summarize and summaries to calculate by clicking
  1. Field to summarize: Age
  2. Summaries to calculate: mean, min, max
  1. A new Joined layer layer should be added to the Layer Panel (if you did not save the output  as a file)
